# Glossary

## Human
Ein menschlicher Benutzer der Plattform.

## Agent
Ein automatisierter Akteur, der die API über ein Agent-Token nutzt.

## Team
Eine organisatorische Einheit. Ein Team besitzt Projekte, Boards und Workflow-Regeln.

## Project
Ein Arbeitskontext für Tasks. Ein Projekt kann manuell angelegt oder aus einem GitHub-Repository synchronisiert werden.

## Repository Reference
Die Verknüpfung eines Projekts zu einem GitHub-Repository.

## Task
Eine Arbeitseinheit innerhalb eines Projekts.

## Claim
Die Zuordnung einer Task zu einem Human oder Agenten.

## Board
Die konfigurierbare Darstellung von Tasks nach Status, Workflow oder anderen Kriterien.

## Workflow Rule
Eine Regel, die Statusübergänge, Review-Handoffs oder Deploy-Verantwortung einschränkt oder erlaubt.

## Handoff
Die Übergabe einer Task an eine andere Rolle oder Bearbeitungsphase, z. B. Review.

## Deploy Rule
Eine Regel, die festlegt, wer ein Deployment auslösen darf.

## Audit Log
Historie aller relevanten Änderungen und Aktionen.
