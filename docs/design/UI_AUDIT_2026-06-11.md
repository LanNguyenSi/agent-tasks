# UI audit digest, 2026-06-11

Point-in-time record of the 7-surface UI/UX audit that produced the overhaul plan in DESIGN_DIRECTION.md. file:line references are valid as of commit dc38db2 on master and will drift as the overhaul lands. 93 findings: 1 critical, 32 high, 49 medium, 11 low.

## dashboard-home
The dashboard works but reads as a functional wireframe rather than a designed product: every element is the same 1px-bordered grey box, status/priority collapse into 7-9px color dots, Unicode arrows stand in for icons, and three stacked control bands (picker row, filter card, duplicate status chips) push actual tasks below ~300px of chrome with no page heading. The deepest structural problem is that 88 inline style blocks in a 1360-line monolith share styling authority with globals.css via !important media-query patches, which makes the restyle the owner wants genuinely risky until BoardView/TaskListView/NewTaskModal/ProjectPicker are extracted and base styles move into classes. Behaviorally, the worst user-facing issues are the shared error state that unmounts the whole loaded board on any transient action failure (with no Retry), and empty states that instruct users to "select a team/project" while offering no control to do so. The home overview is better-engineered (retry, stale-data handling, server counts) but squanders its fetched counts on tiny link labels instead of an at-a-glance stat strip, and duplicates status maps and row markup that should be shared with the dashboard. A redesign pass should sequence: component extraction and token snapping first, then the visual layer (column identity, labeled chips, icon set, type scale), then state/flow fixes.

## tasks-list

## task-detail
The task-detail surface is functionally rich but visually flat and structurally fragile: a 1167-line monolith with 136 inline style objects renders the task title at body-text size under a generic 'Task Details' heading, encodes status as an undifferentiated gray chip, and stacks eight sections in one column while a purpose-built two-column grid class sits dead in globals.css. The highest-impact gaps are workflow integrity (the edit-mode status Select bypasses the gates the advance buttons enforce), an unconfirmed one-click artifact delete, and missing GFM support so agent-written acceptance checklists and tables render as raw text. Consistency debt is heavy: three implementations of the same inline delete control, comments rendered as plain text while sibling fields get markdown, off-scale spacing, and inline color fallbacks that contradict the tokens. States and a11y are partially considered (focus management in the lightbox, keyboard shortcuts) but loading is a bare 'Loading…' despite an existing Skeleton system, and several meanings (activity type, agent authorship, confidence health) are color-only.
