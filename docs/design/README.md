# UI Overhaul

This folder tracks the design-first overhaul of the `agent-tasks` frontend.

## Slice 0/6

This first slice does not change production code yet. It establishes:

- a product-facing design direction
- a UI audit of the current frontend
- low-fi mockups for the primary screens

## Files

- `ui-direction.md` — product principles, visual language, layout system, and rollout plan
- `ui-audit.md` — concrete issues in the current UI, with code references
- `mockups.md` — low-fi screen structures for home, onboarding, teams, dashboard, and task detail

## Intent

The goal is not a decorative reskin. The goal is to make `agent-tasks` feel like a serious operator console for human-agent collaboration:

- clearer hierarchy
- better task scanning
- stronger information density
- fewer "blank form on dark background" moments
- a visual language that feels purposeful instead of placeholder
